Este es el listado de comandos que he aprendido y estoy tratando de dominar

  1. git init

    2. Crea o inicia n nuevo repositorio

  2. git status

    3. Muestra el estado del directorio de trabajo y el area de ensayo y sus ramas

  3. git add

    4. Agrego contenido al archivoindice

  4. git rm --cached

    5. Elimina el archivo del indice, pero lo mantiene en el directorio de trabajo

  5. git rm

    6. Elimina archivos, sin eliminar el historial del sistema de versiones

  6. git reset --soft

    7. Borra el historial y los registros de Git de commits anteriores, pero guarda los cambios en Staging para aplicar las ultimas actualizaciones a un nuevo commit.

  7. git reset --hard

    8. Deshace todo, absolutamente todo. Toda la informacion de los commits y del area de staging se elimina del historial.

  8. git reset --mixed

    9. Borra todo, exactamente todo. Toda la informacion de los commits y del area de staging se elimina del historial.

  9. git reset HEAD

    10. El comando git reset saca archivos del area de staging sin borrarlos ni realizar otras acciones. Esto impide que los ultimos cambios en estos archivos se envien al ultimo commit.
